# How to Test Facebook Ads Creative
**Fuente:** daradenney | https://youtu.be/KFFcno59vQ8

## Qué enseña (2-3 líneas)
Dara Denney explica la metodología exacta que usa el equipo de Thesis para testear creatividades en Facebook Ads, separando los tests en una campaña dedicada, aislando una sola variable por test, usando audiencias broad y escalando ganadores hacia campañas core. También cubre presupuestos, métricas primarias y secundarias para evaluar resultados, y cuándo pausar o escalar.

## Lecciones accionables
- **Estructura de cuenta obligatoria:** Crea una campaña separada para tests creativos con nombre como `t_prospecting_testing_USA`. No mezcles tests con campañas core de prospección y retargeting.
- **Aislar una sola variable por test:** En cada ad set prueba solo un elemento del anuncio: formato (single image, video, carousel, collection, slideshow, GIF), asset net-new (video o imagen), thumbnail, copy (primary text, headline, description) o CTA. No mezcles audiencias, bidding ni estrategias de escalado.
- **Audiencia broad siempre:** Usa ad sets con targeting broad (sin lookalikes ni intereses). Solo aplica guard rails mínimos de edad, género o ubicación geográfica. La razón: el creative dirige el targeting y los resultados más escalables vienen de broad.
- **Nomenclatura de tests:** Asigna un número único cronológico a cada test y un nombre descriptivo del elemento testeado. Ejemplo: `Test_042_Hook_3s_vs_Product_Intro`.
- **Cantidad de ads por ad set:** Entre 3 y 6 anuncios vivos por ad set. Ideal apuntar a 6.
- **Categorías de tests:**
  - **Net new:** Imagen o video nunca antes testeado.
  - **Iterativo:** Una vez identificado un ganador, haz tests iterativos (thumbnail, messaging, copy, CTA) para extender su vida útil.
- **Hook testing para videos nuevos:** Para footage no testeado, prueba diferentes primeros 3 segundos (hook) manteniendo idénticos copy, headline, description y CTA.
- **Copy tests separados:** No uses "multiple text options" dentro del mismo ad porque Facebook no reporta qué copy funcionó mejor. Crea un ad set específico para testear copys.
- **Presupuesto diario mínimo:** Sigue la regla de 50 eventos de optimización por semana para salir de learning phase. Si el CPA promedio es $35, necesitas $250/día ($1,750/semana). En la práctica, no seas militante: si el test rinde mal, pausa antes.
- **% de presupuesto recomendado para testing:** Comienza con 20% del gasto total mensual en la campaña de tests. Ajusta según dónde esté el mejor rendimiento (algunos clientes usan 15%, otros hasta 65%).
- **Evaluación post-lanzamiento:** No analices resultados hasta que hayan pasado al menos 3 días de gasto continuo en el ad set.
- **Tres escenarios después de 3 días:**
  1. **Resultados excelentes:** CPA menor al promedio, 1-2 creativos potenciales. Escala el ad set directamente en la campaña de tests.
  2. **Resultados promedio:** Algunas compras, CPA entre 10-25% del target normal. Monitorea y optimiza a nivel de ad.
  3. **Resultados malos:** CPA alto, pocas o ninguna compra. Pausa el ad set si no hay ganadores después de 5-7 días.
- **Escalado de ganadores:** Antes de mover el creative a campañas core, escala el ad set dentro de la campaña de tests. Incrementa el presupuesto 20% cada 3 días (según best practices de Facebook) o hasta 50-100% si el account lo tolera. Repite 2-3 veces antes de declarar ganador.
- **Nunca apagues un ad set que rinde bien:** Aunque ya hayas duplicado el creative ganador a campañas core, mantén el ad set original activo mientras los resultados sean fuertes. Es una vía de escalado adicional.
- **Optimización a nivel de ad:** Si un anuncio individual tiene CPA alto tras unos días, apágalo para redirigir presupuesto a otros ads. Si un ad alcanza 2x el CPA promedio del account, apágalo.
- **Métricas primarias para evaluar ganadores:**
  - **CPA** (costo por adquisición)
  - **Amount spent** (gasto total): Busca un ad que pueda absorber alto gasto manteniendo CPA bajo.
- **Métricas secundarias para entender el "por qué":**
  - **Hook rate (thumb stop rate):** % de personas que ven los primeros 3 segundos del video.
  - **Hold rate:** % de personas que ven hasta 15 segundos del video.
  - **Unique outbound click-through rate:** % de usuarios únicos que hicieron clic en el CTA y llegaron al sitio web.
  - **Conversion rate (purchases / link clicks):** % de personas que convierten después del clic.
  - **Post shares:** Número de veces que se compartió el anuncio (no optimizar, solo observar).
- **Para marcas nuevas o gasto bajo (<$20k/mes):** Si gastas $5k-$10k/mes, esta metodología puede funcionar pero es mejor simplemente "drippear" (insertar) los nuevos creativos directamente en las campañas core.
- **Orden de prioridad para cuentas nuevas:** Primero haz creative testing para encontrar wins a nivel de ad. Luego, toma esos creativos ganadores y lanza campañas de audience testing.

## Reglas para agentes
- Usa una campaña separada exclusivamente para tests creativos, nunca mezcles tests con campañas core de prospección o retargeting.
- Aísla una sola variable por ad set (formato, hook, copy, thumbnail, CTA, etc.). No combines cambios múltiples en un mismo test.
- Usa siempre audiencia broad (sin lookalikes ni intereses) en los ad sets de testing creativo.
- Asigna un número cronológico único y un nombre descriptivo a cada test.
- Mantén entre 3 y 6 anuncios vivos por ad set de test.
- No uses la herramienta A/B testing de Facebook para tests creativos; duplica costos sin beneficio.
- No analices resultados hasta después de 3 días completos de gasto continuo.
- Si un anuncio individual alcanza 2x el CPA promedio del account, apágalo inmediatamente.
- Nunca apagues un ad set que está generando buenos resultados, incluso si ya duplicaste el creative a campañas core.
- Si no hay ganadores después de 5-7 días, pausa todo el ad set.
- Para escalar un ganador, incrementa el presupuesto del ad set un 20% cada 3 días (o hasta 50-100% si el account lo tolera) y repite 2-3 veces antes de moverlo a campañas core.
- Para cuentas con gasto menor a $20k/mes, prioriza insertar creativos directamente en campañas core en lugar de usar esta metodología completa.

## Errores comunes que evita o menciona
- **No usar una campaña separada para tests:** Mezclar tests con campañas core perjudica el rendimiento general y dificulta el aprendizaje.
- **Probar múltiples variables a la vez:** Impide aislar qué elemento causó el resultado.
- **Usar audiencias segmentadas (lookalikes/intereses) en tests creativos:** Limita la escalabilidad del creative ganador.
- **Usar la herramienta A/B testing de Facebook:** Incrementa costos sin upside real.
- **Diversificar creativos entre prospección y retargeting sin evidencia:** Si algo funciona en prospección, generalmente funciona en retargeting.
- **Apagar ad sets que rinden bien solo porque ya se duplicó el creative:** Se pierde una oportunidad de escalado adicional.
- **Evaluar resultados antes de 3 días:** No hay suficiente distribución de presupuesto ni datos confiables.
- **Ser demasiado rígido con los 50 eventos de optimización:** Si un test rinde mal, pausa antes de alcanzar esa métrica.
- **Usar "multiple text options" dentro de un mismo ad para tests de copy:** Facebook no reporta qué copy funcionó, imposibilitando el aprendizaje.
- **Empezar con audience testing en cuentas nuevas:** Primero se debe encontrar creativos ganadores, luego testear audiencias.
