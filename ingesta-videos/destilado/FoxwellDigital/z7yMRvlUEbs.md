# Account Scaling Call
**Fuente:** FoxwellDigital | https://youtu.be/z7yMRvlUEbs

## Qué enseña (2-3 líneas)
Estrategia para escalar cuentas publicitarias combinando "swings pequeños" (iteraciones rápidas en creativos) con "swings grandes" (nuevas landing pages, tests de precios, ofertas). Enseña a usar un **testing tracker** con codificación por colores para obtener datos direccionales que guíen las próximas pruebas, y a planificar con 2-3 semanas de antelación mediante un calendario de testing.

## Lecciones accionables
- **Usa un testing tracker con columnas clave:** CAC (costo de adquisición), ROAS (retorno sobre inversión publicitaria), "Next Steps" (próximos pasos). Codifica cada test por color (verde = ganador, rojo = perdedor, amarillo = pendiente).
- **Itera sobre tests verdes:** Cuando necesites 5 nuevas ideas para probar, ve a los tests marcados en verde y analiza cómo iterar sobre ellos (ej: cambiar los siguientes 3-5 segundos de un clip UGC ganador, o modificar el text overlay del hook).
- **Crea un "brain splat" semanal:** Una vez por semana, haz una lluvia de ideas basada en el testing tracker y tests pasados. Anota en un documento: idea, estado ("being worked on"), ETA (fecha estimada de finalización), y si es "iteration test" o "net new test".
- **Planifica con 2 semanas de antelación:** Agrega cada idea a un calendario de testing. Ejemplo: si una idea tarda ~1 semana en completarse, programala para testing en 2 semanas. Así sabrás que en 3-4 semanas tendrás datos suficientes para retestar algo nuevo.
- **Prueba variaciones en hooks:** Si un clip UGC funcionó, itera cambiando el hook (text overlay) o los primeros 3-5 segundos del video. No solo el visual; el texto también mueve la aguja.
- **Combina swings pequeños y grandes:** Mientras haces iteraciones pequeñas (cambios en intro, texto overlay), también lanza swings grandes: nuevas landing pages, tests de precios, tests de oferta en la cuenta publicitaria.

## Reglas para agentes
- **Usa el testing tracker con codificación por colores** cuando evalúes resultados de campañas: verde para tests que cumplen o superan el CAC/ROAS objetivo, rojo para los que no.
- **Nunca tomes decisiones solo con datos de 1-2 semanas** para swings grandes; espera a tener datos direccionales de 2-4 semanas antes de retestar o escalar.
- **Siempre programa las pruebas con 2 semanas de anticipación** desde que la idea está lista, para asegurar flujo continuo de tests.
- **Cuando necesites nuevas ideas, prioriza iterar sobre tests verdes** en lugar de empezar desde cero.
- **Mantén un calendario de testing** con columnas: idea, estado, ETA, tipo de test (iteración vs. nuevo), y notas de resultados.

## Errores comunes que evita o menciona
- **No iterar sobre tests ganadores:** Muchos se enfocan solo en crear cosas nuevas y olvidan que los tests verdes son la mejor fuente para iteraciones rápidas que mueven la aguja.
- **No planificar con antelación:** Esperar a que termine un test para pensar en el siguiente causa retrasos. La clave es tener 2-3 semanas de ideas ya calendarizadas.
- **Confundir datos direccionales con datos concluyentes:** El testing tracker da dirección, no verdades absolutas. Úsalo para guiar el próximo paso, no para decidir escalar o matar campañas de forma definitiva.
- **Ignorar los swings grandes:** Hacer solo iteraciones pequeñas (cambios de texto overlay) sin probar cambios estructurales (precios, landing pages, ofertas) limita el crecimiento real de la cuenta.
