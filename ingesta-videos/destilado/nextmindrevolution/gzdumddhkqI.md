# Stop Using Generic Prompts: Mastering 5 AI Styles based on Digital, Pixels & Digital.
**Fuente:** nextmindrevolution | https://youtu.be/gzdumddhkqI
## Qué enseña (2-3 líneas)
Enseña cómo dejar de pedir estilos genéricos y controlar resultados visuales digitales usando palabras clave técnicas.  
Cubre glitch art, pixel art por bits, one-bit art, ASCII art y voxel art, principalmente en Nano Banana Pro dentro de Google Flow.

## Lecciones accionables
- Usa Google Flow con Nano Banana Pro para probar estilos visuales; también menciona que se pueden usar herramientas como Hex Field, Polo o Magnific.
- Para exploración inicial, prueba prompts mínimos como: `glitch art style`, `8 bit art style`, `16 bits`, `ASCII art style` o `voxel art`.
- Para glitch art, distingue entre glitch analógico y glitch digital. Si solo escribes `glitch art style`, el modelo puede devolver cualquier mezcla de error analógico, error digital, pixelación o corrupción de datos.
- Para glitch analógico, especifica explícitamente `analog`, `analog tape`, `VHS type of glitch` o `analog glitch`; si no lo haces, Nano Banana tiende a devolver glitch digital con `data corruption` y `pixelation`.
- Para glitch digital, usa keywords exactas como `digital pixel sorting`, `sharp macro blocking`, `data corruption` y `RGB channel splitting`.
- Prompt descrito para glitch digital: una imagen de paisaje con montaña usando `digital pixel sorting`, `sharp macro blocking`, `data corruption` y `RGB channel splitting`.
- Para pixel art, no uses solo `pixel art` si necesitas una estética concreta. Especifica `8-bit`, `16-bit` o `32-bit` porque cada uno cambia el nivel de detalle.
- `8-bit` produce menos detalle y una estética más retro; `16-bit` añade más detalle; `32-bit` puede acercarse a una estética más avanzada.
- El video aclara que `4-bit` no corresponde como estilo de consola en este contexto.
- Si pides `Sega style`, `8 bits`, `16 bits` o `32 bits` en Nano Banana, puede generar una interfaz de videojuego o pantalla de consola, no necesariamente una ilustración artística.
- Para obtener pixel art más artístico en Nano Banana, añade instrucciones visuales explícitas como `minimal`, `highly artistic`, `visual`, colores concretos y sujeto concreto.
- Prompt descrito para pixel art minimalista: `minimal eight bit pixel art` de un atardecer en la playa con una palmera, usando colores `pink` y `cyan`.
- Prompt verbatim mencionado: `A detailed pixel art portrait of an aesthetic woman with violet hair wearing a retro-futuristic jacket`.
- Para el retrato pixel art, añade `retro pixel cyberpunk style`, `low-fi color palette`, `highly artistic` y `visual` para evitar que Nano Banana lo vuelva demasiado realista.
- Para one-bit art, usa la keyword `one bit`. El estilo se inspira en Macintosh temprano y Game Boy, y trabaja con solo dos colores.
- Para one-bit fiel al estilo, usa combinaciones como `dark and white` o `dark green and light green`; si el verde sale demasiado saturado, pide un verde más pálido.
- Prompt descrito para one-bit: `high contrast moody cyberpunk`, `one bit`, colores más fieles al estilo, y remate con `artistic` y `aesthetic`.
- Para ASCII, usa `ASCII art style`; si escribes solo `ASCII`, el resultado puede ser aburrido o demasiado literal.
- ASCII art construye imágenes usando caracteres imprimibles. Puedes limitar caracteres concretos para controlar el resultado.
- Prompt descrito para ASCII: una mujer en estilo ASCII con `glowing green text on the background`, `visual`, `artistic` y `retro`.
- Para restringir ASCII, usa solo caracteres específicos como `@`, `#`, `.`, `/`.
- Para resultados ASCII más expresivos, permite más caracteres y añade colores; el video destaca que mezclar ASCII con color genera resultados más artísticos que solo blanco y negro.
- Para voxel art, usa `voxel art`; voxel significa `volumetric pixel`, básicamente un pixel 3D convertido en cubo.
- Si solo pides `voxel art`, Nano Banana puede devolver algo parecido a Minecraft: cubos en espacio tridimensional, pero con acabado simple.
- Para voxel más artístico, usa prompts tipo: `A detailed voxel art diorama of a lush floating island at sunset. Complex cube geometry, soft volume lighting, miniature aesthetic, highly artistic`.

## Reglas para agentes
- Usa `analog`, `analog tape` o `VHS type of glitch` cuando el usuario quiera glitch de cinta, VHS o error analógico.
- Usa `digital pixel sorting`, `sharp macro blocking`, `data corruption` y `RGB channel splitting` cuando el usuario quiera glitch digital.
- Nunca uses solo `glitch art style` si el resultado debe ser específicamente analógico o específicamente digital.
- Usa `8-bit`, `16-bit` o `32-bit` cuando el usuario pida pixel art retro; no uses solo `pixel art` si el nivel de detalle importa.
- Usa `minimal eight bit pixel art` cuando el usuario quiera pixel art simple, retro y de baja resolución.
- Añade `highly artistic` y `visual` en Nano Banana cuando el modelo tienda a generar pantallas, consolas o escenas realistas en vez de una imagen artística.
- Usa `one bit` cuando el usuario pida estética Game Boy, Macintosh temprano, dos colores o minimalismo extremo.
- Limita la paleta de one-bit a dos tonos verificables: oscuro/blanco, oscuro/verde claro o verde oscuro/verde claro.
- Usa `ASCII art style` cuando el usuario quiera imágenes hechas con caracteres; no uses solo `ASCII`.
- Restringe caracteres ASCII con una lista explícita, por ejemplo `@`, `#`, `.`, `/`, cuando el usuario quiera control tipográfico.
- Usa `voxel art` cuando el usuario quiera pixels 3D, cubos, dioramas cúbicos o estética tipo Minecraft.
- Para voxel artístico, añade sujeto, composición y acabado: `diorama`, `complex cube geometry`, `soft volume lighting`, `miniature aesthetic`, `highly artistic`.

## Errores comunes que evita o menciona
- Pedir estilos genéricos como `glitch art style` sin indicar si el glitch debe ser analógico o digital.
- Esperar que `pixel art` produzca siempre el mismo resultado, cuando `8-bit`, `16-bit` y `32-bit` tienen niveles de detalle distintos.
- Usar `4-bit` como si fuera una categoría equivalente a las consolas retro mencionadas.
- Pedir `8 bits`, `16 bits` o `Sega style` en Nano Banana y sorprenderse de recibir una interfaz de videojuego en vez de una pieza artística.
- No añadir `highly artistic` o `visual` en Nano Banana cuando se quiere una imagen estética y no una escena realista.
- Escribir solo `ASCII` en vez de `ASCII art style`, lo que puede producir resultados poco interesantes.
- No especificar caracteres en ASCII cuando se necesita control formal.
- Pedir solo `voxel art` y aceptar un resultado básico tipo Minecraft, en vez de enriquecerlo con `diorama`, iluminación, geometría y estética miniatura.
