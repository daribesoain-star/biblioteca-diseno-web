# How I Actually Approach Targeting in 2026
**Fuente:** JonLoomer | https://youtu.be/WfoA-2u5KEA

## Qué enseña (2-3 líneas)
Jon Loomer explica su enfoque minimalista y basado en datos para la segmentación en Meta Ads en 2026, eliminando casi todas las opciones tradicionales de targeting. La estrategia se centra en dejar que el algoritmo optimice libremente, usando solo controles de ubicación, exclusiones y reglas de valor para corregir problemas demográficos específicos.

## Lecciones accionables
- **No usar audience suggestions**: ignora completamente esta opción; no aporta valor ni perjudica, pero es irrelevante para la estrategia.
- **No restringir por edad o género**: cuando se optimiza para compras (purchase), el algoritmo encuentra a los compradores correctos automáticamente. Si hay un problema demográfico (ej. baja calidad de leads en cierto rango de edad), aplicar **value rules** para pujar menos en ese grupo, no restringir.
- **Eliminar detailed targeting y lookalike audiences**: ambas opciones quedaron obsoletas porque Meta ya tiene datos históricos y en tiempo real. Además, para objetivos de rendimiento (conversiones, leads) solo funcionan como "sugerencias", no como restricciones reales.
- **No aislar remarketing con custom audiences**: Meta ya asigna naturalmente 20-25% del presupuesto a remarketing sin necesidad de segmentarlo. Verificar con el desglose (breakdown) por "audience segments".
- **Configuración de ubicaciones**: agrupar países similares en un mismo conjunto de anuncios (ad set) a menos que el volumen justifique separarlos. Ejemplo: EE.UU., Canadá, Reino Unido y Australia juntos.
- **Excluir custom audiences solo para evitar que quienes ya compraron vean el anuncio**: no excluir todas las audiencias personalizadas (no es "prospecting puro"), solo aquellas que ya realizaron la acción promocionada. Aceptar que las exclusiones no son perfectas y no preocuparse por ello.
- **Regla de valor para leads**: si se ejecutan campañas de leads, aplicar una value rule para ajustar por edad (age adjustment) cuando sea necesario.

## Reglas para agentes
- No uses audience suggestions nunca; son irrelevantes.
- No restrinjas por edad o género cuando optimices para purchase; si hay un problema demográfico, aplica value rules para reducir puja en ese grupo.
- No uses detailed targeting ni lookalike audiences para objetivos de rendimiento (conversiones, leads); solo funcionan como sugerencias.
- No aísles remarketing con custom audiences a menos que sea un caso excepcional y raro; Meta ya lo hace automáticamente (~20-25% del presupuesto).
- Agrupa países similares en un mismo ad set a menos que el volumen justifique separarlos.
- Excluye custom audiences solo para evitar que quienes ya compraron vean el anuncio; no excluyas todas las audiencias personalizadas.
- Cuando ejecutes campañas de leads, aplica una value rule para ajustar por edad si hay problemas demográficos.

## Errores comunes que evita o menciona
- **Obsesionarse con el control del targeting**: restringir demasiado perjudica los resultados; el enfoque "hands-off" es más efectivo.
- **Usar audience suggestions como parte importante de la estrategia**: no hacen nada y solo generan confusión.
- **Restringir por edad/género para resolver problemas de calidad de lead o LTV**: la solución correcta es usar value rules para pujar menos, no eliminar grupos.
- **Aislar remarketing manualmente**: Meta ya prioriza remarketing sin inputs; hacerlo manualmente es redundante.
- **Usar detailed targeting o lookalikes como si fueran restricciones reales**: en objetivos de rendimiento solo son sugerencias, no segmentaciones efectivas.
- **Excluir todas las custom audiences en campañas de prospección**: solo se debe excluir a quienes ya realizaron la acción promocionada para evitar desperdicio.
- **Preocuparse por la imperfección de las exclusiones**: las exclusiones no son perfectas, pero no vale la pena perder el sueño por ello.
