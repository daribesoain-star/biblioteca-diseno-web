# Woman disguises 911 call by ordering pizza
**Fuente:** camp_value | https://youtu.be/ZJL_8kNFmTI

## Campana(s) y marca(s) analizadas
No es una campaña publicitaria tradicional. Es un caso de comunicación en crisis real donde una víctima de violencia doméstica usó un código encubierto (pedir pizza) para alertar a emergencias sin que el agresor lo notara. Se analiza como **arquetipo de comunicación encriptada** aplicable a marcas.

## Insight humano (la verdad incómoda/universal debajo)
**"Cuando estás en peligro real, no puedes pedir ayuda con palabras directas. Necesitas un código que el agresor no entienda, pero que el salvador sí."**
La incomodidad: en situaciones de coerción, la víctima debe ser más astuta que el victimario. La comunicación directa es un lujo que no todos pueden pagar.

## Estructura / mecanismo narrativo
**Ley aplicada: #4 Literalizar metáfora**
La metáfora "pedir ayuda disfrazada" se literaliza: la persona realmente pide una pizza, pero cada elemento del pedido es un código (tamaño = tipo de emergencia, dirección = ubicación exacta). El drama está en que el espectador descubre el código después de ver la escena "normal".

**Estructura:**
1. **Setup engañoso:** Escena cotidiana (pedido de pizza).
2. **Fricción sutil:** Tono de voz tenso, pausas extrañas, repeticiones.
3. **Revelación:** El operador de 911 explica el código.
4. **Cierre emocional:** La víctima fue rescatada gracias a ese código.

## Por qué funciona / qué la hace VENDER
- **Tensión narrativa:** El espectador siente que algo está mal pero no sabe qué, hasta que se revela el código. Eso genera **engagement emocional**.
- **Empoderamiento:** La víctima no es pasiva; usa inteligencia para salvarse. Esto inspira y vende la idea de que "con astucia, puedes ganar incluso cuando estás perdiendo".
- **Memorabilidad:** El código es simple pero ingenioso. La gente lo recuerda y lo comparte (efecto viral).
- **Universalidad:** Cualquier persona puede imaginar estar en esa situación. No es un caso raro, es un miedo común.

## Cómo se aplicaría a un foodservice B2B (carne por caja, ángulo "sé vivo, no pagues caro / B2Vio")

**Campaña sugerida:** *"El código de la carne"*

**Mecanismo:** Usar el mismo principio de comunicación encriptada, pero para que el comprador B2B (carnicerías, restaurantes, distribuidores) detecte sobreprecios ocultos o engaños en facturación.

**Ejecución concreta:**
- **Anuncio de 30 segundos:** Un carnicero llama a su proveedor actual y pide "una caja de lomo, pero con el código 911". El vendedor se confunde. El carnicero cuelga y le explica a su hijo: "Cuando pido 'código 911', significa que me están cobrando 30% más que en B2Vio. Es mi señal para cambiar de proveedor."
- **Código real:** En cada factura de B2Vio, incluir un "código de verificación" que el comprador puede usar para comparar precios en tiempo real. Si el código no coincide con el precio del mercado, es señal de que el otro proveedor está abusando.
- **Ángulo B2Vio ("sé vivo, no pagues caro"):** "No necesitas gritar que te están robando. Solo necesitas el código correcto. Pide B2Vio y recibe tu código de verificación. Si tu proveedor actual no te da el mismo, ya sabes qué hacer."

**Por qué funciona en B2B:**
- El comprador B2B se siente **empoderado** (como la víctima del 911) frente a proveedores que lo engañan.
- El código es **discreto** (no confrontación directa, solo verificación).
- La marca B2Vio se posiciona como **aliado inteligente**, no como vendedor agresivo.
- **Accionable:** El comprador puede llamar a su proveedor actual y pedir "el código 911" como prueba de lealtad. Si no lo recibe, cambia.

**Frase final de campaña:**
*"No pagues caro. Pide el código. Pide B2Vio."*
