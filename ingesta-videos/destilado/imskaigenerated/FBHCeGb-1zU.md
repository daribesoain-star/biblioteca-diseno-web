# ChatGPT vs Claude vs Gemini Make an AI Film From One Prompt
**Fuente:** imskaigenerated | https://youtu.be/FBHCeGb-1zU
## Qué enseña (2-3 líneas)
Compara ChatGPT, Gemini y Claude dentro de Higgsfield Supercomputer usando el mismo brief para crear un corto de acción AI de 45-60 segundos. Muestra cómo generar guion, shot list, prompts para Seedance, imagen de personaje consistente y edición final en CapCut.
## Lecciones accionables
- Usa un solo brief estructurado para todo el pipeline: pedir al modelo que actúe como “expert short film director” y cree un “complete 45- to 60-second AI action film from scratch”, incluyendo “script, shot list, and detailed Seedance prompts”, manteniendo “one consistent character throughout the story”.
- Flujo base: abrir Higgsfield Supercomputer, elegir ChatGPT/Gemini/Claude, pegar el mismo prompt, generar guion + planos + prompts, copiar el prompt de personaje, ir a Image, elegir Nano Banana 2, pegar prompt, poner aspect ratio `16:9`, generar imagen de referencia.
- Para video: ir a Video, cambiar modelo a Seedance/SeaDance, pegar el prompt del plano, subir la imagen de referencia, etiquetarla en el prompt, definir duración y generar.
- Para continuidad entre planos: subir siempre la imagen de referencia más el último video generado, etiquetarlos en el prompt y repetir los rasgos del personaje.

## Reglas para agentes
- Usa el mismo prompt base cuando compares modelos; no cambies el brief entre ChatGPT, Gemini y Claude.
- Genera primero una imagen de personaje antes de crear video.
- Usa Nano Banana 2 para crear la referencia visual del personaje cuando trabajes dentro de este flujo.
- Usa aspect ratio `16:9` para la imagen de referencia si el corto será horizontal.
- En Seedance, sube la imagen de referencia en todos los planos.
- Después del primer plano, sube también el último clip generado para preservar continuidad visual.
- Repite rasgos clave del personaje en cada prompt: rostro, cicatriz, ropa, accesorio principal, mano/muñeca correcta y color dominante.
- Define estilo visual explícito: paleta, iluminación, género, clima, lente o cámara si aplica.
- Define audio en el prompt cuando el plano dependa de atmósfera: lluvia, vapor, murmullo, zumbido de dron, alarmas.
- Usa secciones separadas tipo Claude cuando el plano sea complejo: estilo, narrativa, movimiento, elementos estáticos, cámara, reparto/set y audio.
- Usa `dynamic description` para acción y movimiento; usa `static description` para entorno, vestuario y objetos persistentes.
- Mantén un objeto identificable durante todo el corto si ayuda a la historia: maletín, data drive, implante, chaqueta crimson.
- Da una conclusión narrativa verificable; no termines solo con una imagen espectacular.
- Exporta y une los clips en CapCut solo después de generar todos los planos.
## Errores comunes que evita o menciona
- Intentar escribir manualmente cada prompt y encadenar planos sin una estructura central.
- No fijar una imagen de personaje antes de pasar a video.
- Cambiar detalles del personaje entre planos y perder consistencia.
- No subir el clip anterior como referencia para el siguiente plano.
- Crear visuales fuertes sin cierre narrativo, como ocurrió con Gemini.
- Mezclar acción, entorno y cámara en un bloque confuso en vez de separarlos por función.
- Olvidar parámetros básicos como `16:9`, duración del plano o calidad `1080p`.
- Depender solo del look cyberpunk/neón sin una progresión clara de historia.
- No repetir los rasgos del personaje en cada prompt.
- No revisar que el objeto principal permanezca visible y consistente, como el maletín o la chaqueta crimson.
