# remuneraciones_imposiciones

## Qué enseña (2-3 líneas)
Este manual explica cómo generar desde el módulo de Remuneraciones de KAME dos archivos clave para cumplir obligaciones laborales: la Declaración de Imposiciones para Previred y el Libro de Remuneraciones Electrónico (LRE) para la Dirección del Trabajo. Detalla los pasos exactos para obtener cada archivo y las validaciones previas necesarias para evitar rechazos.

## Lecciones accionables
- **Generar Declaración de Imposiciones (Previred):**
  - Ingresar al módulo de **Remuneraciones**.
  - Hacer clic en **"Acciones Masivas"** y seleccionar **"Generar Previred"**.
  - Indicar el **mes** que se desea declarar.
  - Hacer clic en **generar** para descargar un **archivo plano**.
  - Subir ese archivo a la sesión de **Previred** para revisar montos y realizar el pago.
- **Verificar cuadratura antes de pagar:**
  - Usar el **Informe de Cuadratura de Previred** dentro del sistema para verificar que toda la información coincida.
- **Generar Libro de Remuneraciones Electrónico (LRE):**
  - Dentro del mismo módulo de **Remuneraciones**, seleccionar **"Acciones Masivas"** → **"Generar LRE"**.
  - Seleccionar el **periodo** y descargar el archivo.
- **Validar formato de fechas del LRE:**
  - Antes de subir el archivo a la Dirección del Trabajo, revisar que las fechas en el Excel estén en formato **DD/MM/AAAA**.
  - Ajustar el formato si el computador local no coincide con el exigido por la plataforma de la DT.

## Reglas para agentes
- Usa **"Acciones Masivas" → "Generar Previred"** cuando necesites generar la declaración de imposiciones para el pago de leyes sociales.
- Usa **"Acciones Masivas" → "Generar LRE"** cuando necesites informar datos de trabajadores a la Dirección del Trabajo.
- Siempre ejecuta el **Informe de Cuadratura de Previred** antes de subir el archivo plano a Previred para verificar montos.
- Siempre revisa que las fechas en el archivo LRE estén en **formato DD/MM/AAAA** antes de subirlo a la DT.
- Nunca subas el archivo LRE a la Dirección del Trabajo sin verificar el formato de fechas, para evitar rechazos o errores de validación.

## Errores comunes que evita o menciona
- **Formato de fechas incorrecto en LRE:** El formato del computador local puede no coincidir con el formato DD/MM/AAAA que exige la plataforma de la Dirección del Trabajo, causando rechazos o errores de validación. Se recomienda ajustar las fechas antes de subir el archivo.
- **Falta de verificación de cuadratura:** No usar el Informe de Cuadratura de Previred antes de pagar puede llevar a errores en los montos declarados.
