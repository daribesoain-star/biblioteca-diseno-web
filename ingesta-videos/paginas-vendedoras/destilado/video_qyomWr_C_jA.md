# The Easy Way to Design Top Tier Websites
**Fuente:** https://youtu.be/qyomWr_C_jA | destilado 2026-07-22

## Qué enseña (2-3 líneas)

El video desmitifica el diseño web de alto nivel como un proceso sistemático de conexión de ideas, no de inspiración repentina. Enseña 5 reglas fundamentales (simplicidad, leyes Gestalt, espaciado, sistemas de diseño, jerarquía) y un método de 5 pasos para crear sitios que convierten, usando ejemplos concretos de apps financieras y benchmarks de espaciado.

## Lecciones accionables

- **Regla 1: "Good design is as little design as possible"** — No empieces por el header ni la estructura general. Pregúntate: "¿Cuál es la funcionalidad clave o el principal punto de venta?" Para muchos sitios es: un heading, un input field y un botón. Diseña solo eso primero.
- **Regla 2: Ley de similitud y proximidad** — Usa forma, tamaño, color y espaciado para agrupar elementos. El cerebro procesa primero el todo, luego los detalles. El diseño debe ser escaneable en segundos.
- **Regla 3: "Elements need more spacing than you think"** — Empieza con mucho espaciado (ej. 40px) y luego reduce hasta que te guste el resultado. Los usuarios escanean la UI completa antes de enfocarse en elementos individuales.
- **Regla 4: Sistema de diseño con valores divisibles por 4** — Usa esta escala de espaciado (en px): 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96, 128. Convierte a REM dividiendo entre 16 (ej. 40px → 2.5rem). Asigna como variables globales.
- **Tipografía: una sola fuente + type scale** — No hay ciencia exacta para colores: elige un color oscuro y uno claro para texto/fondo, más dos colores de acento para personalidad. Asegura legibilidad, no abrumes.
- **"Avoid text to line center, especially for paragraphs and smaller text size"** — Texto centrado solo para títulos cortos, nunca para párrafos.
- **Line height inversamente proporcional al font size** — Texto más pequeño necesita mayor line height. El line height también actúa como margen superior automático entre elementos de texto.
- **Botones y links: dos tipos cada uno** — Primarios y secundarios. Define ambos en tu sistema de diseño.
- **Regla 5: "Hierarchy is everything"** — Para enfatizar un elemento: primero reduce contraste de información secundaria, luego aumenta font weight, luego font size. Zoom out para verificar que el título destaque.
- **"Not all H1 tags will have the same size and margins"** — El contexto manda. A veces un H3 o un párrafo necesita más tamaño que un H2.
- **Excepciones a la regla de simplicidad** — Introduce profundidad con sombras (reemplazan bordes sólidos), colores de acento, gradientes sutiles, y cards para elementos planos.
- **Proceso creativo en 5 pasos:**
    1. Conoce las bases (reglas + libros recomendados: no se nombran títulos específicos en el video)
    2. Encuentra inspiración: sitios top mundiales, Figma Community, Mobbin (app de referencias de diseño). Ejemplo concreto: para una sección de testimonios de app financiera → filtrar en Mobbin por "testimonial section" + categoría "finance".
    3. Trabaja mentalmente sobre los diseños: analiza qué funciona como usuario. Para fintech: "human face + simple language", evitar "generic testimonial sections with no emotions".
    4. Aléjate del problema: "watch few tutorials or read some articles, then take a break and do something else". Si no fluyen ideas nuevas, revisa si hay estrés o falta de sueño.
    5. "Don't fall in love with it" — Prueba con amigos/colegas, luego con usuarios. Ajusta según feedback. "Sometimes you have to design several tier websites just to discover that you designed one good pricing section in the third design."

## Reglas para agentes

- **Usa la escala de espaciado divisible por 4 (px) convertida a REM** cuando definas márgenes, paddings y gaps entre componentes.
- **Empieza con 40px de espaciado entre secciones** y reduce progresivamente hasta encontrar el punto óptimo visual.
- **Nunca diseñes con Lorem Ipsum o datos vagos** — el espaciado que funciona para una card puede ser desastroso para otra.
- **Asigna line height mayor cuanto menor sea el font size** — para texto de 14-16px usa line-height de 1.5-1.6; para títulos de 32px+ usa 1.1-1.2.
- **Usa color de acento solo para elementos que requieren atención inmediata** — botones primarios, CTAs, enlaces clave.
- **Reemplaza bordes sólidos con sombras** cuando quieras que un elemento se sienta más cercano al usuario y atraiga más foco.
- **Aplica gradientes sutiles** en lugar de colores sólidos planos para añadir "excitement" sin romper la simplicidad.
- **Para jerarquía: primero reduce contraste de elementos secundarios, luego aumenta weight, luego size** — en ese orden, no al revés.
- **Zoom out después de diseñar** para verificar que el título principal destaque del contenido secundario.
- **Cuando estés atascado, aléjate del problema** — mira tutoriales, lee artículos, luego haz otra actividad. No fuerces la creatividad bajo estrés.

## Errores comunes

- **Empezar por el header o la estructura general** ("how many sections do I need, how wide they should be, how should I design the buttons") — esto drena la creatividad. Empieza por la funcionalidad clave.
- **Usar demasiados elementos** que frustran usuarios y se ven feos. El cerebro busca simplificar; más diseño = diseño más feo.
- **Diseñar con espaciado insuficiente** — lo que parece "demasiado espacio" para el diseñador es correcto para el usuario que escanea la UI completa.
- **Usar valores de espaciado aleatorios** en lugar de un sistema consistente (divisible por 4).
- **Centrar texto en párrafos o textos pequeños** — solo funciona para títulos cortos.
- **Caer en tutoriales de "psicología del color"** — no hay ciencia real; solo elige colores legibles que no abrumen.
- **Enamorarse del primer diseño** — prueba con otros, muestra a colegas, itera. "Finish something, anything. Stop planning and thinking to design and just design it."
