# Cómo comenzar a DISEÑAR una WEB o APP
**Fuente:** https://youtu.be/4Azc4Gs2nlU | destilado 2026-07-22

## Qué enseña (2-3 líneas)

Este video explica por qué el diseño y prototipado deben preceder a la programación, y muestra cómo usar Miro para crear wireframes funcionales en minutos. Enseña a transformar ideas abstractas en prototipos visuales que ahorran tiempo, dinero y re-trabajo durante el desarrollo de apps o sitios web.

## Lecciones accionables

- **Nunca empieces programando:** El creador (Mauri, 12+ años como ingeniero de software) advierte que programar sin diseño previo genera "hacer y rehacer" constantemente, desperdiciando tiempo y dinero. El orden correcto es: análisis → diseño/prototipado → programación.
- **Usa Miro para prototipado rápido:** Registro gratuito en miro.com, sin límite de usuarios. Funciona en web, Windows, Mac, iOS y Android. La versión gratuita es suficiente para proyectos individuales.
- **Aprovecha las plantillas de wireframe:** Dentro de Miro, busca "wireframe" en la barra de búsqueda de plantillas. Encontrarás componentes pre-armados para apps móviles, navegadores y tablets. Solo arrastra y suelta.
- **Usa la librería "WiFi Library":** Al hacer clic en esta opción dentro de Miro, aparecen componentes prediseñados (frames, botones, campos de texto, imágenes) que puedes seleccionar y arrastrar directamente al canvas.
- **Sigue guías de alineación:** Miro muestra guías visuales automáticas para centrar elementos, alinear y mantener espaciado consistente entre componentes. Esto permite seguir "reglas de diseño sólidas" sin ser diseñador.
- **Prototipa en 1 minuto:** El ejemplo del video muestra cómo diseñar una pantalla tipo YouTube (imagen de cabecera, avatar, título, descripción, botón suscribir, lista de videos) en aproximadamente 60 segundos.
- **Usa "Frames" para organizar pantallas:** Cada pantalla o vista se representa como un "Frame" en la barra lateral. Puedes renombrarlos (ej: "Canal") y agruparlos dentro de un "ship" (contenedor) para navegar entre ellos.
- **Vincula pantallas entre sí:** Puedes conectar frames para simular navegación (ej: seleccionar un video → ir a otra pantalla). Esto permite visualizar el flujo completo de la app.
- **Añade notas y comentarios:** Miro permite agregar notas adhesivas, comentarios y anotaciones a nivel de código directamente sobre el prototipo, facilitando la comunicación con desarrolladores.
- **Exporta el prototipo:** Una vez refinado, puedes exportarlo para continuar con una fase de refinamiento en herramientas de diseño más avanzadas.

## Reglas para agentes

- **Usa Miro (o herramienta similar) para prototipar ANTES de escribir cualquier línea de código.** El diseño y la programación no deben ocurrir simultáneamente.
- **Define primero el MVP (Producto Mínimo Viable):** "Es mejor crear algo pequeño y funcional que un producto muy grande cuya dirección no conocemos". Enfoca todo el diseño en esa versión mínima.
- **Usa plantillas de wireframe prediseñadas cuando existan.** No reinventes la rueda; las plantillas aceleran el proceso y siguen convenciones de UX probadas.
- **Mantén espaciado consistente entre elementos.** Usa las guías de alineación automáticas de la herramienta para asegurar uniformidad visual.
- **Nombra cada Frame con el nombre de la pantalla o vista que representa** (ej: "Canal", "Inicio", "Perfil"). Esto organiza el prototipo y facilita la navegación entre pantallas.
- **Simula la navegación vinculando frames entre sí** para validar el flujo de usuario antes de programar.
- **Incorpora imágenes reales e iconos** en el prototipo para acercarlo al producto final y tomar decisiones visuales tempranas.
- **Añade notas y comentarios a nivel de código** directamente sobre el prototipo para documentar decisiones técnicas.
- **Exporta el prototipo refinado** antes de pasar a la fase de programación. No programes hasta que el diseño esté congelado.

## Errores comunes

- **Programar antes de diseñar:** El error más grave. Lleva a "hacer y rehacer" constantemente, desperdiciando tiempo y dinero porque se está diseñando y programando al mismo tiempo.
- **Subestimar la importancia del diseño y la usabilidad:** Una app técnicamente impecable pero con mal diseño o UX hará que el usuario decida no usarla. "No importa qué tan bien esté implementado el código".
- **Saltar directamente a la implementación sin un análisis previo:** No tener claros el enfoque principal de la aplicación ni el MVP genera productos sin dirección clara.
- **Usar un cuaderno físico para prototipar:** Dificulta la organización, encontrar elementos y hacer cambios (hay que tachar y redibujar constantemente). Las herramientas digitales como Miro permiten cambios rápidos y ordenados.
- **Ignorar el diseño por no ser diseñador:** El creador (ingeniero de software con 12+ años) insiste en que los programadores deben interesarse por UX. "Tener conocimiento sobre UX nos ayudará a crecer como profesionales". No hace falta ser diseñador para prototipar.
